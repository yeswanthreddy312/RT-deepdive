Wavefront .obj File Format

Fundamentals Reference — Structure, Data Types, and Parser Conventions

1. What Is a Wavefront .obj File?

A Wavefront .obj file is a plain-text, human-readable 3D geometry format originally developed by Wavefront Technologies. It is one of the most universally supported interchange formats for 3D meshes: every major 3D tool (Blender, Maya, 3ds Max) can export it, and it requires no binary parsing, making it an ideal first target for a ray tracing engine loader.

Each line of an .obj file begins with a keyword prefix that declares the type of data on that line. Lines beginning with # are comments and are ignored by parsers.

Minimal valid .obj file (a single triangle): 
# A minimal triangle mesh
v   0.0  1.0  0.0
v  -1.0 -1.0  0.0
v   1.0 -1.0  0.0
f  1 2 3

2. Core Data Types

A ray tracing engine needs only three of the many keywords the format supports. The table below documents them.

Prefix Name Syntax Description
v Geometric Vertex v x y z A point in 3D space. The three floats are the X, Y, and Z coordinates. Accumulated into the vertex list in the order they appear in the file.
vn Vertex Normal vn nx ny nz A unit (or non-unit) normal vector at a vertex, used for smooth shading. Not necessarily normalised by the exporter — the parser or shader must normalise if needed.
f Face Definition f v1 v2 v3 or f v1//vn1 v2//vn2 v3//vn3 Defines a polygonal face by referencing vertex (and optionally normal) indices. See Section 3 for the critical indexing rule.

2.1 Geometric Vertex (v)

Vertices are the atomic building blocks of the mesh. Each v line adds one point to the vertex list. The order of v lines in the file defines the implicit 1-based index of each vertex — the first v line is vertex 1, the second is 2, and so on. 

# Four vertices of a tetrahedron
v   1.0  1.0  1.0   # index 1
v  -1.0 -1.0  1.0   # index 2
v  -1.0  1.0 -1.0   # index 3
v   1.0 -1.0 -1.0   # index 4

2.2 Vertex Normal (vn)

Normal vectors are stored in a separate, parallel list to the vertex list. They are referenced independently inside face definitions using the v//vn syntax. A normal index of 1 refers to the first vn line in the file. 

vn   0.577  0.577  0.577   # normal index 1
vn  -0.577 -0.577  0.577   # normal index 2

2.3 Face Definition (f)

A face line references vertices (and optionally normals) by their list index. For a triangle, exactly three index groups are listed. The two most common syntaxes you will encounter: 

# Vertex indices only
f  1 2 3

# Vertex // Normal indices (no texture coordinate)
f  1//1  2//2  3//1

3. The 1-Based Index Convention — Critical Implementation Note

⚠ Off-by-One Error: The Most Common .obj Parser Bug

The .obj format uses 1-based indexing: the first vertex in the file has index 1, not 0. Python lists and NumPy arrays use 0-based indexing: the first element is at position 0. Failing to convert means every face lookup is shifted by one position, causing silent geometry corruption or an IndexError on the final vertex.

The conversion rule is simple and must be applied to every vertex and normal index read from an f line:

$$\text{python\_index} = \text{obj\_index} - 1$$
Example — parsing a face line and applying the index shift: 
# .obj file line:   f  3  7  12
#                       ↓  ↓   ↓   subtract 1 from each
# Python indices:       2  6  11

parts = line.split()          # ["f", "3", "7", "12"]
face  = [int(p) - 1 for p in parts[1:]]   # [2, 6, 11]
faces.append(face)

The same shift applies when the face uses the v//vn syntax — both the vertex index and the normal index must each have 1 subtracted: 

# .obj file line:   f  1//2  3//1  5//2
# Each token is split on "//" to separate vertex and normal refs

for token in parts[1:]:
    v_idx, vn_idx = token.split("//")
    vertex_indices.append(int(v_idx)  - 1)
    normal_indices.append(int(vn_idx) - 1)

4. Annotated Full Example — Tetrahedron

A complete .obj file for a tetrahedron, with every line annotated:

# Tetrahedron — 4 vertices, 4 triangular faces

# ── Vertices (1-based: v1 … v4) ──────────────────────────────
v    1.0  1.0  1.0
v   -1.0 -1.0  1.0
v   -1.0  1.0 -1.0
v    1.0 -1.0 -1.0

# ── Normals (outward-facing per face) ────────────────────────
vn   0.577  0.577  0.577
vn  -0.577 -0.577  0.577
vn  -0.577  0.577 -0.577
vn   0.577 -0.577 -0.577

# ── Faces (1-based vertex//normal refs) ──────────────────────
f   2//1  3//1  4//1    # face 1 — opposite vertex 1
f   1//2  4//2  3//2    # face 2 — opposite vertex 2
f   1//3  2//3  4//3    # face 3 — opposite vertex 3
f   1//4  3//4  2//4    # face 4 — opposite vertex 4

Parser Checklist

  • Skip lines starting with # (comments) or blank lines.
  • Dispatch on the first token: v, vn, f.
  • On every index read from an f line, apply index - 1 before storing.
  • Normalise vn vectors if smooth shading is required (exporters are not guaranteed to emit unit normals).
